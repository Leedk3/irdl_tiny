1. udp통신  
	udp통신: 인터넷에서 정보를 주고받을 때 사용하는 통신 규약(프로토컬) 중 하나
	- 상대방이 받았는지 여부 중요x 일단 데이터를 보냄 (대비: tcp통신)
	
	udp통신 특징
		- 비연결형: 테이터를 보내기 전 미리 연결을 맺는 과정x (주소를 알면 데이터 송부)
		- 속도가 빠름: 확인 절차가 없음 통신이 빠르고 부하가 적음
		- 신뢰성 낮음: 데이터가 중간에 유실되거나 순서가 바뀌어도 책임x
		- 일대 다 통신 가능: 한 명이 여러 명에게 데이터를 뿌리는 브로드캐스트나 멀티 캐스트가 가능

	왜 이러한 방식을 쓰나?
	> 데이터가 불안정 하여 새더라도 실시간 성이 중요한 분야 
	- 화상통화: 지금 화면이 잠깐 화질이 깨져도 다음 연결이 중요
	- 온라인 게임: 지금 화면이 멈춰도 그 다음 데이터가 실시간으로 실행되는 게 중요 

2. 기본 용어 정리
	- socket(데이터를 보내주는 길): IP와 Port가 결합된 출입구
	- stream(정보가 연속적/순차적으로 전송): 끊기지 않는 데이터의 흐름
	- packet(데이터를 담는 공통 포장지): 네트워크 장비(라우터)가 인식하는 단위
	- segment(데이터를 담는 기준): 번호가 매겨진 상자 (신뢰성-tcp통신)
	- datagram(데이터를 담는 그릇): 독립적인 봉투 (속도-udp통신)

3. udp통신 및 tcp통신 비교
	- udp통신: socket를 통해 데이터가 datagram의 형태로 packet에 담겨져서 들어오는 데로 데이터 처리
		> datagram 형태로 packet 끼리 연결x 오는 순서대로 그냥 뜯어봄 

	- tcp통신: socket를 통해 데이터가 segment의 형태로 packet에 담겨져서 순서에 맞게 데이터 처리 
		> segment 형태로 순서가 적혀 있어 순서에 맞게 뜯어봄 

* 1bit = (0,1) 
  1byte = 8bits
  bit > byte > KB > MB > GB > TB *10^3

4. 통신 데이터 처리 과정
	- 데이터를 보낼수 있는 최대 크기 MTU(Maximum Transmission Unit) = 1500byte

	- 데이터 캡슐화 과정
		1) 테이터 생성(ex. 로봇의 위치 좌표)
		2) udp datagram: 8byte udp 헤더(봉투 안내서) + 테이터(로봇의 위치 좌표)
		3) ip packet: 20byte ip 헤더(소포 안내서) + udp datagram

5. udp datagram
	- 데이터(ex.로봇의 위치(1.3))의 메세지(payload)에 8Byte의 헤더를 붙인 봉투
		> 8Bytes 헤더? 데이터란 봉투가 어디로 가야할지에 대한 안내서 
	
	- 8Byte udp 헤더의 구조
		> Source Port(16bits): 데이터를 보내는 프로그램의 포트 번호 = 보내는 문
		  Destination Port(16bits): 데이터를 받을 프로그램의 포트 번호 = 받는 문
		  Length(16bits): 헤더와 데이터를 합친 전체 길이로 어디까지가 한 패킷인지 알려줌 = 메세지 길이
		  Checksum(16bits): 전송 중 데이터가 깨지지 않았는지 확인하는 최소한의 안전장치 = 오류 검사

6. ip packet 
	- 데이터 정보(payload)와 udp헤더가 담긴 udp datagram에 ip 헤더를 붙힌 소포
	- 20byte ip 헤더의 구조
		> Version(4bits): IP 버전(보통 IPv4인 '4'가 들어감)
		  TTL(Time To Live, 8bits): 패킷의 생명력 라우터를 하나 거칠 때마다 1씩 줄고 0이 되면 패킷은 자동 폐기(무한 루프 방지)
		  Protocol(8bits): 상자 안에 든 내용물이 무엇인지 알려줌 UDP는 17번, TCP는 6번이라는 코드를 사용
		  Source IP Address(32bits): 보내는 기기의 IP 주소
		  Destination IP Address(32bits): 받는 기기의 IP 주소 네트워크 장비(라우터)들이 가장 중요하게 보는 정보

7. 'ip 헤더' VS 'udp 헤더' 
	  구분   |     IP 헤더 (소포 주소판)     |   UDP 헤더 (봉투 안내서)   |
	  크기   |    최소 20바이트 (무거움)     |    딱 8바이트 (매우 가벼움) |
	핵심 역할 |    컴퓨터 위치(집) 찾기       |    프로그램 위치(방) 찾기   |  
	주요 주소 | IP 주소 (예: 192.168.0.10)  | 포트(Port) 번호 (예: 8080)|  
	  계층   |     네트워크 계층 (Layer3)   |     전송 계층 (Layer4)    |

8. 고정 ip를 쓰는 이유? 
	- UDP는 데이터가 잘 갔는지 확인X: TCP는 "주소 맞나?"라고 물어보고 통신을 시작하지만, UDP는 그냥 주소가 적힌 소포(IP packet)를 던지고 끝남
		> 만약 유동 IP로 IP가 바뀌면 데이터를 계속 던져도 없는 주소로 감
	- 통신의 지연 방지:	유동 IP는 일정 시간마다 주소를 다시 할당받는 과정에서 통신이 아주 미세하게 끓김
		> 주소 할당 과정을 생략해 통신의 즉각성 확보
	- 다수 기기의 식별: 여러 대의 로봇이 협력할 때, 192.168.0.21은 1호기, 192.168.0.22는 2호기라는 식으로 논리적인 설계 가능
		
	- 고정 ip VS 유동 ip
		> 고정 ip(Static IP): 사용자가 직접 (192.168.0.10)처럼 주소를 딱 박아버리는 방식 아무리 껐다 켜도 주소가 변함 X 
		> 유동 ip(DHCP): 기기가 켜질 때마다 공유기가 남는 주소를 빌려주는 방식(오늘 주소와 내일 주소가 다를 수 있음)
